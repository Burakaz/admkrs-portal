E-Mail-Signatur in 5 Minuten.

Trag unten deine Daten ein, kopiere den fertigen Code und setze ihn nach der Anleitung in Apple Mail ein. Der Generator erzeugt exakt die offizielle ADMKRS-Signatur — inklusive Dark-Mode-Support und ohne Umlaut-Probleme. Zum Mitschauen gibt es weiterhin das Loom-Video (alter Ablauf — den Signatur-Code nimmst du jetzt von hier).

01

Deine Daten

Alles andere (Adresse, Logo, Icons) ist fest hinterlegt — du füllst nur diese fünf Felder.

Mit Leerzeichen, wie es angezeigt werden soll — der Anruf-Link wird automatisch daraus gebaut.

Live-Vorschaufolgt deinem System-Design
02

Code kopieren

Sonderzeichen werden automatisch abgesichert (HTML-Entities) — der Code funktioniert dadurch in jeder .mailsignature-Datei.

Fülle erst alle fünf Felder aus.

Der Code erscheint hier, sobald alle Felder ausgefüllt sind.
03

In Apple Mail einsetzen

Voraussetzung: Dein admkrs-Google-Konto ist in Apple Mail eingerichtet.

TextEdit vorbereiten (einmalig)

  • TextEdit öffnen (⌘ + Leertaste → „TextEdit").
  • Einstellungen → Neues Dokument → Haken bei „Reiner Text".
  • Einstellungen → Öffnen und Sichern → Haken bei „HTML-Dateien als HTML-Code anzeigen".
  • TextEdit schließen.

Platzhalter-Signatur in Mail anlegen

  • Mail → Einstellungen → Signaturen → +.
  • Signatur „ADMKRS" nennen und als Text nur blaaaaaaaaaaaaaa eintragen (daran erkennst du gleich die richtige Datei).
  • Haken bei „Immer meine Standardschrift verwenden" entfernen (Wortlaut je nach macOS-Version leicht anders).
  • Die Signatur per Drag & Drop deinem admkrs-Konto zuweisen und beim Konto als Signatur „ADMKRS" auswählen; Haken bei „Signatur über zitiertem Text platzieren" setzen.
  • Mail komplett beenden (⌘Q) — sonst überschreibt Mail deine Änderung gleich wieder.

Signatur-Datei öffnen

Terminal öffnen (⌘ + Leertaste → „Terminal") und diesen Befehl ausführen:

open -a TextEdit ~/Library/Mobile\ Documents/com\~apple\~mail/Data/V*/Signatures/*.mailsignature

Falls „no matches found" erscheint (kein iCloud Drive), stattdessen:

open -a TextEdit ~/Library/Mail/V*/MailData/Signatures/*.mailsignature

Es öffnen sich eine oder mehrere Dateien in TextEdit.

Code einsetzen

  • Finde die Datei mit dem blaaaaaaaaaaaaaa.
  • Die obersten Zeilen bis einschließlich „Mime-Version: …" bleiben stehen.
  • Alles darunter (inkl. blaaaaaaaaaaaaaa) löschen und deinen kopierten Code einfügen.
  • Speichern (⌘S), TextEdit schließen.
  • Ohne iCloud Drive (zweiter Befehl)? Dann die Datei jetzt schützen: im Finder auswählen → ⌘I → Haken bei „Geschützt" — sonst überschreibt Mail sie beim nächsten Start.
Content-Transfer-Encoding: 7bit
Content-Type: text/html; charset=us-ascii
Message-Id: <XXXXXXXX-XXXX-…>
Mime-Version: 1.0 (Mac OS X Mail …)
<html> … dein kopierter Code … </html>

Die Zeilen können bei dir leicht abweichen (z. B. steht das charset in einer eigenen Zeile) — entscheidend ist nur: alles bis einschließlich „Mime-Version" bleibt stehen.

Testen

  • Mail öffnen → neue E-Mail → Signatur „ADMKRS" prüfen (Name, Telefon-Link, LinkedIn).
  • Schick dir selbst eine Test-Mail — auch einmal in den Dunkelmodus wechseln.

Warum bleibt der Header stehen?

Die Zeile Message-Id muss zur ID im Dateinamen der .mailsignature passen — beides vergibt Mail beim Anlegen der Signatur. Deshalb lässt sich der Header nicht mitliefern: Er ist pro Mac und pro Signatur einzigartig. Ersetzt wird immer nur alles unterhalb von „Mime-Version".

Mail hat die Signatur nach einem Neustart überschrieben? Dann in dieser Reihenfolge: 1. Mail beenden (⌘Q). 2. Befehl aus Schritt 03 erneut ausführen und deinen Code noch einmal einsetzen + sichern. 3. Erst jetzt die Datei schützen: open ~/Library/Mail/V*/MailData/Signatures öffnet den Ordner im Finder → Datei auswählen → ⌘I → Haken bei „Geschützt". Danach Mail starten.

Beide Befehle melden „no matches found"? Prüfe, dass die Platzhalter-Signatur aus Schritt 02 wirklich angelegt ist und Mail beendet wurde. Hilft das nicht, gib Terminal Festplattenvollzugriff (Systemeinstellungen → Datenschutz & Sicherheit → Festplattenvollzugriff), starte Terminal neu und wiederhole den Befehl.

TextEdit zeigt die Datei hübsch gerendert statt als Code? Schritt 01 (Einstellungen) prüfen und die Datei erneut öffnen.
